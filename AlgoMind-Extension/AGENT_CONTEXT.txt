SECTION 3 — KEY ARCHITECTURE DECISIONS
==============================================================================

1. COMMUNICATION FLOW
   React App (window.postMessage)
       ↓
   content/algomind.js (chrome.runtime.sendMessage)
       ↓
   background.js (state machine + API)
       ↑
   background.js (chrome.tabs.sendMessage → content/algomind.js → window.postMessage)
       ↑
   React App (window.addEventListener 'message')

2. EXTENSION DETECTION (two-method fallback)
   a) DOM Sentinel: content script injects <div id="__algomind_extension_installed__">
      React checks via document.getElementById() — instant, zero latency
   b) postMessage Ping/Pong: React sends ALGOMIND_PING, script replies ALGOMIND_PONG
   Timeout: 1500ms → if no response, extensionInstalled = false

3. HEARTBEAT (MV3 constraint workaround)
   - chrome.alarms minimum is 1 minute in production MV3
   - Self-rescheduling setTimeout loop (4s) inside the service worker
   - chromes.alarms fires once/min as a wakeup in case service worker sleeps

4. STATE PERSISTENCE
   - All state in chrome.storage.local under key 'algomind_state'
   - On service worker startup, state restored and heartbeat resumes if challenge active

5. VIOLATION SYSTEM (3-Strike Rule)
   Triggers:
     - Tab switch to non-AlgoMind / non-platform tab
     - New tab created during challenge
     - Window focus lost (chrome.windows.onFocusChanged → WINDOW_ID_NONE)
     - Navigation to restricted paths (LeetCode /solutions, /discuss; CF /blog, /tutorial, /gym)
   Strike 1 → Warning notification
   Strike 2 → Final warning notification
   Strike 3 → ALGOMIND_TRIGGER_FORFEIT postMessage → React auto-calls backend forfeit API

6. RANKED PARTY UI
   - ChallengesPage home view has a "Ranked Code Party" section below casual party
   - The entire section (Host Ranked Party + Join buttons) is wrapped in <ExtensionGuard>
   - If extension absent: shows shield icon + "Install Extension" CTA (no buttons visible)
   - If extension present: shows Host Ranked Party and Join input/button

SECTION 4 — MESSAGE TYPES (complete reference)
==============================================================================

React App → Extension (window.postMessage):
  ALGOMIND_CHALLENGE_START  { type, partyCode: string }
  ALGOMIND_CHALLENGE_END    { type }
  ALGOMIND_PING             { type }

Extension → React App (window.postMessage via algomind.js relay):
  ALGOMIND_PONG             { type, installed: true }
  ALGOMIND_HEARTBEAT        { type, timestamp: number }
  ALGOMIND_VIOLATION        { type, strikes: number, reason: string }
  ALGOMIND_TRIGGER_FORFEIT  { type, reason: string }
  ALGOMIND_EXTENSION_STATUS { type, active: boolean, strikes: number }

Internal (BG ↔ content scripts via chrome.runtime.sendMessage):
  ALGOMIND_CHALLENGE_START       → background starts monitoring
  ALGOMIND_CHALLENGE_END         → background stops monitoring
  ALGOMIND_PING                  → background responds with PONG
  ALGOMIND_STATE_REQUEST         → background returns { state: {...} }
  ALGOMIND_PLATFORM_VIOLATION    → platforms.js → background (violation from platform tab)

SECTION 5 — REACT INTEGRATION (how to use)
==============================================================================

A) Extension detection + Ranked Party guard:
   import ExtensionGuard from './components/ExtensionGuard/ExtensionGuard';

   <ExtensionGuard>
     <button onClick={joinRankedParty}>Join Ranked Party</button>
   </ExtensionGuard>

B) Hook into challenge lifecycle (already done in ChallengesPage.jsx):
   import { dispatchChallengeStart, dispatchChallengeEnd, listenToExtension }
     from './hooks/useAlgoMindExtension';

   // When host clicks "Start Party":
   dispatchChallengeStart(partyCode);

   // When challenge ends / forfeit / leave:
   dispatchChallengeEnd();

C) Auto-forfeit listener (already wired in ChallengesPage.jsx):
   useEffect(() => {
     if (view !== 'room' || !partyCode) return;
     return listenToExtension((msg) => {
       if (msg.type === 'ALGOMIND_TRIGGER_FORFEIT') {
         handleForfeit(msg.reason); // calls backend /party/<code>/forfeit/
       }
     });
   }, [view, partyCode]);

SECTION 6 — BACKEND FORFEIT API
==============================================================================

Endpoint : POST /challenges/party/<code>/forfeit/
Auth     : IsAuthenticated (bearer token)
Logic    : Sets member.finished_at = now, calls _assign_ranks(party)
File     : Backend/challenges/views.py — class ForfeitPartyView (line 314)
URL      : Backend/challenges/urls.py — path('party/<str:code>/forfeit/', ...)

The frontend calls this via:
  api.post(`/challenges/party/${partyCode}/forfeit/`, { reason })
